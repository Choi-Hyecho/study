01. 자바 프로그래밍 시작하기

프로그래밍 : 컴퓨터에게 일을 하도록 명령어를 만드는 것
 -> 내가 원하는 기능, 내가 원하는 단계로 컴퓨터가 일을할 수 있도록 하는거 
 
컴파일 : 프로그래밍 언어를 컴퓨터가 알 수 있는 언어로 바꿔주는 일
 -> 컴퓨터는 0과 1로만 소통 할 수 있음. 각 언어 마다 맞는 컴파일러가 필요함. 

Java -> 처음에는 임베디드 쪽으로 개발됨. 백색 가전을 위해서 개발되었음 이전에 C나 C++이 있었지만 pointer를 사용해서 오류가 많았음.
자바는 객체 지향적 언어 oop orient object programing

다른 언어를 쓸 수 있는 사람은 2장 부터 하라고 하셨지만

나는 새로운 마음으로 하나하나 탐구 하면 공부해야하니까 그냥 쭉 들어야지....

컴파일러 : 기계어로 번역해주는 프로그램

JRE   Java runtime env
-> 자바가 돌아가기 위해서는 자바가 돌 수 있는 환경이 필요함. 
JRE는 지원하는 최신 버전 다운
JDK는 버전 노상관

이클립스나 인텔제이 설지

자바를 사용하면 좋은 이유

자바소스 코드 -> 컴파일러 -> 바이트 코드 -> 그래서 컴파일할 때 오류가 남

C 언어와 자바 언어가 다른점

C는 OS마다 컴파일이 들림 리눅스, 윈도우 컴파일러가 따로따로 있음.
그래서 각각 컴파일 하면 윈도우 실행파일 리눅스 실행파일 따로임
그대신 자바 보다 빨랐음

자바는 컴파일러 돌리면 바이트 코드로 변환 되며
자바 가상 환경 JRE만 깔려 있으면 어디서든 윈도우 리눅스 OS 상관 없이 아무데서 쓸 수있음.
옛날엔 C보다 실행속도가 많이 느렸지만 이제 가상환경도 좋아지고 기계도 좋아져서 속도 차이는 의미가 없어짐.

자바는 객체 지향 언어이기 때문에 유지보수가 쉽고 확장성이 좋음
C나 C++ 이후에 나오면서 개선되어 나왔음.

프로그램이 안정적임

풍부한 기능을 제공하는 오픈 소스임.

스크립트 언어 -> 실행되면서 번역됨. 스크립터 인터트랩터 번역이 되면서 동시실행 그래서 실행할때 오류남
